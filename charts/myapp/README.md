# values.yaml — параметры для настройки Helm-чарта

## Основные параметры

- `tag`:  
  Определяет тег Docker-образа, который будет использован. Например: `latest` или `v1.0.0`.

- `replicas`:  
  Количество реплик вашего приложения. По умолчанию — 1.

## Конфигурация образа

- `image.repository`:  
  Полный путь к вашему Docker-образу без тегов. Например: `hub.docker.com/skillfactory/testapp`.

- `image.tag`:  
  Тег Docker-образа. Обычно связывается с переменной `tag`, которую можно менять.

## Сервис

- `service.type`:  
  Тип службы Kubernetes: `ClusterIP`, `NodePort` или `LoadBalancer`.  
  Для публичного доступа чаще всего используют `LoadBalancer` или `NodePort`.

- `service.port`:  
  Внешний порт сервиса, по которому будет доступно приложение (обычно 80).

- `containerPort`:  
  Порт внутри контейнера, на котором работает ваше приложение. В вашем случае — 8000.

---

## Как использовать

1. Откройте файл `values.yaml`.
2. Отредактируйте нужные параметры.
